Keycloak Client Authentication Scenarios

This app demonstrates Keycloak client authentication scenarios using client secrets, Kubernetes service-account tokens, and SPIFFE JWT-SVID assertions.

Client authentication using SPIFFE JWT-SVID machine to machine

Backend reads a JWT-SVID minted by SPIRE for this workload and sends it as client_assertion with Keycloak's SPIFFE assertion type.

participant "SPIRE Server" as Server
participant "Keycloak" as KC

group #E3F2FD JWT-SVID Fetch [Workload API]
  App -> Agent : fetch jwt-svid (aud=Keycloak realm)
  Agent -> Server : attest workload + check registration entry
  Server --> Agent : issue JWT-SVID (sub=spiffe://example.org/myclient)
  Agent --> App : JWT-SVID
end

group #E8F5E9 Token Request [SPIFFE grant]
  App -> KC : POST /token (client_credentials + jwt-spiffe assertion)
  KC -> Server : GET SPIFFE bundle endpoint
  Server --> KC : JWT bundle keys
  KC -> KC : Validate JWT-SVID signature + claims
  KC --> App : Access Token
end

App -> App : Decode and display JWT

@enduml

Pseudo code

jwt_svid = read("/var/run/secrets/spiffe/jwt-svid.json")

POST /token
  grant_type=client_credentials
  client_assertion_type=jwt-spiffe
  client_assertion=jwt_svid

access_token = response.access_token
